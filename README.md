# TheHuddle-Desafio2-Codigo_Heredado

# ¿Cómo dividiste las responsabilidades?
Dividi las responsabilidades separando la logica del mapa, el calculo de rutas y la interfaz grafica en Clases Independientes
Para que cada parte del algoritmo tenga una funcion clara y hace que el codigo sea mas facil de entender

# ¿Qué aprendiste del proceso de refactorización?
Aprendi sobre la importancia de organizar el codigo de forma modular
Aprendi sobre POO y la importancia de este tipo de programacion en caso de que en el futuro se necesite algun tipo de mantenimiento o actualizacion
Aprendi sobre los 4 pilares fundamentales de POO y para que nos sirve cada uno

# ¿Qué decisiones tomaste y por qué?
Una de las decisiones principales; Decidir en cuantas clases estructurar mi codigo, al principio crei que dos eran suficientes, pero luego me recomendaron tener toda la parte que maneja la interfaz grafica (Tkinter) en una clase separada

Estas decisiones se tomaron para que el codigo sea mas facil de modificar sin romper otras partes del algoritmo
